从字符到字形：传统蒙古文 Unicode 编码

传统蒙古文数字化核心：码位、控制符、变体选择器、字体与排版引擎如何共同完成正确显示。

先看图示 看开发者说明

一句话说清楚

传统蒙古文 Unicode 不是“一个视觉字形一个码位”。它保存的是名义字符序列，最终看到的首形、词中形、词尾形、变体与竖排效果，由控制符、字体和 shaping engine 共同决定。

核心公式

字符编码 + 控制符 + OpenType 字体 + 排版引擎 = 正确显示

总览图：从输入到最终显示

1

输入 / 键盘输入

输入法输出的是字符序列，不应该直接输出“看起来像”的图片或私有码位。

2

Unicode 码点序列

3

控制符 / 辅助符号

FVS1 U+180BFVS2 U+180CFVS3 U+180DMVS U+180ENNBSP U+202FZWJ U+200D
4

Shaping Engine / 排版引擎

根据上下文决定独立形、词首形、词中形、词尾形，并处理变体选择。

5

字体 / OpenType 字体

字体提供具体字形轮廓与替换规则，例如 GSUB/GPOS。

6

最终显示

ᠮᠣᠩᠭᠣᠯ

竖排：从上到下，列从左到右。

Unicode 区块概览

  • U+1800–U+180A：标点符号
  • U+180B–U+180D：FVS1–FVS3
  • U+180E：MVS（历史兼容与新规范相关）
  • U+1810–U+1819：数字
  • U+1820–U+1842：蒙古字母
  • U+1843–U+18AA：扩展字母与附加符号

为什么特别？

  • 文字竖排，不是普通横排脚本
  • 同一字母会随位置改变字形
  • 同一字符可有不同变体
  • 正确显示依赖字体 + 排版引擎

常见误区

  • Unicode ≠ 字体
  • 编码的是字符，不是图片或固定字形
  • 看到一样，不代表底层码点一样
  • 缺少字体时可能显示为方框或错形

关键判断：
1. 数据库保存码点序列，不保存截图。
2. 输入法输出标准字符，不输出私有编码。
3. 字体必须支持 Mongolian OpenType shaping。
4. 网页竖排优先测试 writing-mode: vertical-lr。
5. 搜索、复制、粘贴、排序、表单输入都要单独验收。

1. 基础码位

每个字符都有 Unicode 编号。例如：ᠠ = U+1820，ᠨ = U+1828，ᠭ = U+182D。

2. 变体与控制符

FVS 用于请求特定字形变体；ZWJ/ZWNJ 辅助连接；MVS/NNBSP 涉及分写与历史兼容问题。

3. 渲染机制

文本先保存为码点序列，引擎读取上下文并选形，字体输出正确连写字形。

4. 实践要点

不要把传统蒙古文当图片处理。编码、字体、排版、输入法四者必须一起测试。
