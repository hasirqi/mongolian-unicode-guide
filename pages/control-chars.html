Control Characters

FVS / MVS / NNBSP / ZWJ

这些“看不见或不容易看见”的字符，是传统蒙古文正确显示、连接和变体选择的关键。

控制符为什么重要？

传统蒙古文不是单纯把字母排在一起就能正确显示。某些场景需要变体选择器或连接控制符，帮助排版引擎选择正确字形。

注意

控制符通常不可见。看起来一样的文本，底层码点序列可能不同。

常见控制符与辅助字符

名称码位用途风险
FVS1U+180B请求第一类自由变体。字体或引擎不支持时可能无效。
FVS2U+180C请求第二类自由变体。不同字体表现可能不同。
FVS3U+180D请求第三类自由变体。需要测试具体字符。
MVSU+180E历史兼容相关字符。规范和实现历史复杂，使用需谨慎。
NNBSPU+202F常用于后缀前的窄不换行空格。复制、换行、搜索时要测试。
ZWJU+200D零宽连接符，辅助连接和取形。滥用会造成不可预期显示。
ZWNJU+200C零宽不连接符，阻断连接。可能影响搜索和排序。

FVS

FVS 不代表新字符，而是告诉系统：这个字符在这里希望使用某个标准化变体。

ZWJ / ZWNJ

用于影响连接关系。它们看不见，但会改变 shaping 结果。

NNBSP

常见于后缀处理，是传统蒙古文文本处理里绕不开的测试点。

开发者必须提供的调试视图

显示文本：ᠮᠣᠩᠭᠣᠯ
底层码点：U+182E U+1823 U+1829 U+182D U+1823 U+182F

建议工具：
- 显示每个字符的 Unicode 编号
- 高亮不可见字符
- 显示 FVS / ZWJ / NNBSP 的位置
- 支持复制码点序列
- 支持不同字体对比

实用原则

控制符不是“越多越好”。优先保存标准文本，只有在确实需要指定变体或连接状态时使用控制符，并用测试用例验证。
