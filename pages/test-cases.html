Test Cases

测试用例与验收清单

把“显示不对”变成可以复现、可以提交 Issue、可以修复的工程测试。

测试目标

传统蒙古文支持不能只看“页面上似乎显示了字”。真正的验收要覆盖输入、存储、复制、搜索、竖排、字体、浏览器、PDF 和图片导出。

测试原则

每个测试项都要记录：输入文本、底层码点、字体、浏览器、系统、截图、预期结果、实际结果。

基础测试矩阵

测试项要验证什么通过标准
输入输入法是否输出标准 Unicode 字符序列。码点序列与预期一致。
显示独立形、词首形、词中形、词尾形是否正确。目标字体与标准样例一致。
控制符FVS、ZWJ、ZWNJ、NNBSP 是否被正确处理。不可见字符没有丢失，显示符合预期。
竖排writing-mode、列方向、标点方向是否正确。从上到下，列从左到右，标点不乱。
复制粘贴复制后底层码点是否保持。粘贴结果与原始码点完全一致。
搜索排序搜索是否能找到同一文本，排序是否稳定。不因字体或显示形态影响文本操作。
导出PDF、图片、打印是否保持显示。导出结果与网页显示一致。

建议测试样例

样例 1：基础词
显示：ᠮᠣᠩᠭᠣᠯ
码点：U+182E U+1823 U+1829 U+182D U+1823 U+182F

样例 2：位置形测试
目标：测试同一字母在独立、词首、词中、词尾的显示差异
记录：输入序列 / 截图 / 字体 / 浏览器

样例 3：控制符测试
目标：测试 FVS1/FVS2/FVS3、ZWJ、ZWNJ、NNBSP 是否保留并影响显示
记录：可见文本 + 不可见字符位置

样例 4：竖排测试
CSS：writing-mode: vertical-lr; text-orientation: mixed;
目标：检查列方向、标点方向、复制粘贴结果
